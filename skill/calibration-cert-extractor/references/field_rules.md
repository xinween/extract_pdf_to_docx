# 字段格式规则

## 必填字段（顺序固定）

| 序号 | 字段名 | 说明 |
|------|--------|------|
| 1 | 序号 No. | 自动编号，从1开始 |
| 2 | 仪器仪表名称 Instruments Name | 英文在上，中文在下 |
| 3 | P&ID编号 P&ID No | 读取不到则留空 |
| 4 | 序列号 Series No. | 器具编号/序列号 |
| 5 | 品牌 Brand | 英文在上，中文在下 |
| 6 | 型号 Model | 完整型号 |
| 7 | 校准证书编号 Calibration Certificate No. | 证书唯一编号 |
| 8 | 校准日期 Cal. Date | YYYY.MM.DD 格式 |
| 9 | 有效期至 Due Date | YYYY.MM.DD 格式 |
| 10 | 备注 Remark | **一律留空** |

## 格式规则

### 1. 双语字段格式

仪器仪表名称和品牌字段，如果同时存在英文和中文：
- **英文在上**
- **中文在下**
- 使用手动换行符 `\n` 连接

示例：
```
Electric contact pressure gauge
电接点压力表
```

### 2. 日期格式

- 统一使用 `YYYY.MM.DD` 格式
- 例如：2026.01.21

### 3. 有效期计算

优先级：
1. **优先读取**证书上的"下次校准日期"
2. 如无，则按校准日期 + 12个月 - 1天计算

### 4. 备注字段

- **一律留空**
- 不填写任何内容

### 5. 温度探头特殊规则

读取温度探头证书时：
- **使用管理编号替代器具编号**
- 注意区分单支/双支温度探头

## 数据结构

数据以 JSON 格式传入脚本：

```json
{
  "no": "1",
  "name_en": "Electric contact pressure gauge",
  "name_cn": "电接点压力表",
  "pid": "",
  "sn": "M0315769",
  "brand_en": "ZY",
  "brand_cn": "",
  "model": "(0~16)bar",
  "cert": "AIMT2026-F-012299",
  "cal_date": "2026.01.21",
  "due": "2027.01.20"
}
```

## 读取原则

- **只提取证书真实存在的信息**
- 读不到/不确定的字段必须留空
- **绝对不编造**
- 不跳页、不省略
